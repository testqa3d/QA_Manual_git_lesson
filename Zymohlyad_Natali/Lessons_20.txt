git clone <URL> — створення локальної копії віддаленого репозиторію (наприклад, з GitHub).

git status — перевірка стану файлів: які змінені, а які ще не відстежуються.

git add <файл> або git add . — додавання змін до індексу (підготовка до збереження).

git commit -m "Повідомлення" — збереження знімка стану коду з коротким описом (коміт).

git pull — завантаження змін з віддаленого сервера та їх об'єднання з вашим поточним кодом.

git push — надсилання локальних збережених комітів на віддалений сервер.
